# Engineering rules for the Stride backend

## Stack lock-ins
- pnpm only. Do NOT run `npm` or `yarn` — they break the lockfile.
- Node 20 LTS pinned in `.nvmrc`; do not bump without RFC.
- Prisma is the only DB access path. Exceptions: `src/legacy/` (being retired) and `src/db/raw/` (escape hatch).

## Forbidden patterns
- No `as any` in TypeScript. Use `unknown` and narrow.
- No mocking the database in integration tests (see `/docs/testing.md` for why).
- No `--no-verify` on commits. Pre-commit hooks are load-bearing.
- No extending `TOKEN_CACHE` in `src/middleware/auth.ts` — deprecated; use `SessionStore`.
- No `-- TODO` in migration Down blocks — breaks rollback tests; the CI check rejects it.

## Conventions you can't see from the code
- Files under `src/generated/` are autogenerated by `pnpm gen` — never edit.
- HTTP handlers are thin: parse → call service → return. No business logic in handlers.
- All Prisma writes go through the transaction wrapper in `src/db/tx.ts`.
- Custom errors are in `src/lib/errors.ts` (`AppError`, `NotFoundError`, `ValidationError`). Don't throw bare `Error`.
