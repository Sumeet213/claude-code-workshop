M3 — Surgery debrief

CLAUDE.md surgery — what to cut and why

The bloated before_CLAUDE.md is 130 lines. It’s not obviously bad — every section was added in good faith, and most teams have something like it committed today. That’s the trap.

The trimmed version is 35 lines. Here’s what came out, and why a senior engineer should defend each cut even when their colleagues object.

The cuts, by category

❌ Project history and “lessons learned” prose (~20 lines cut)

“We originally built on Sequelize but migrated to Prisma in Q3 2024 after the third major migration footgun…”

“We tried microservices in 2023 and rolled it back six months later…”

“The session store moved from in-process LRU to Redis after a P0 in November 2024…”

Why cut: Project history belongs in commit messages, RFCs, and the /docs/rfcs/ directory you already linked. Inlining it costs tokens on every turn — the model reads the Sequelize war story when you ask it to format a date.

What to push back on if a colleague defends it: “Will the model do something demonstrably wrong if it doesn’t know we used to use Sequelize?” Almost always no. The history is for humans onboarding, not for the agent loop.

❌ Architecture overview prose (~15 lines cut)

The src/api/, src/services/, src/db/ walkthrough.

Why cut: The model can run ls src/ and see this in one tool call, on demand, when it actually matters. Inlining it pays the cost on every turn for context the model uses on maybe 5% of tasks.

What to keep instead: one line linking to a fuller architecture doc. “Architecture overview: /docs/architecture.md (read on demand).” Lazy loading.

❌ “How to develop” tutorial (~15 lines cut)

The pnpm install, pnpm dev, pnpm test table.

Why cut: All of those are in package.json scripts, which the model reads when it needs them. The CLAUDE.md is duplicating documentation that’s already authoritative elsewhere.

What to keep instead: the lock-ins that aren’t visible from package.json — “pnpm only. npm and yarn will break the lockfile.” That’s a load-bearing rule the model can’t infer.

❌ Tutorials disguised as conventions (~25 lines cut)

“Adding a new endpoint: 1. Create the handler in src/api… 2. Register the route in src/app.ts… 3. Add types…”

“Adding a database migration: 1. Edit prisma/schema.prisma… 2. Run pnpm prisma migrate dev…”

Why cut: These are how-to guides for humans. The model can read existing endpoints and follow the pattern by example. Inlining tutorials is the most common bloat in real-world CLAUDE.md files.

What to keep instead: the anti-pattern the tutorial is hinting at — “do not leave -- TODO in the Down block of a migration; it breaks rollback tests.” Specific. Loadbearing. The tutorial implied this rule but buried it on line 7 of step 4.

❌ Empty platitudes (~6 lines cut)

“Write good code. Be careful with security.”

“Handle errors.”

“Write tests for new features.”

Why cut: The model already wants to do these. Saying “be careful with security” doesn’t make it more careful — it just costs tokens. Replace with specific anti-patterns (“no as any”, “no mocking the DB in integration tests”).

Easy giveaway: any sentence that starts with “please” or “try to” is almost certainly cuttable.

❌ Style rules already enforced by tooling (~5 lines cut)

“2-space indentation, single quotes, semicolons. Use descriptive variable names. Use named exports.”

Why cut: prettier and eslint are authoritative. If the model emits wrong style, the format-on-write hook fixes it. Don’t pay token cost on every turn for what runs once at save time.

❌ Volatile state (~8 lines cut)

“We are in Sprint 23. STR-432 (Priya, in review). STR-441 (Diego, blocked). The next planning meeting is Thursday 2pm IST.”

Why cut: True today, wrong next month. This is standup notes, not load-bearing rules. Project memory or your issue tracker holds volatile state.

❌ Credential location prose (~5 lines cut)

“The production DATABASE_URL is in AWS Secrets Manager under stride/prod/db.”

Why cut: Even pointing at where credentials live narrows attacker search if this file ever leaks. The model can ask if it needs to know.

❌ Useful-links section (~6 lines cut)

The Grafana, Papertrail, Sentry URLs.

Why cut: These are reference material. Belongs in a top-level LINKS.md or your team wiki — not in the file the model reads on every turn.

What we kept (and why each line earned its place)

Kept Why it’s load-bearing
“pnpm only. npm/yarn break the lockfile.” Model defaults to npm; this stops it.
“Files under src/generated/ are autogenerated. Never edit.” Genuine invariant the model can’t infer from the code.
“Handlers should be thin (parse → call service → return).” Architectural rule; contradicts the obvious “put logic in handler” Express default.
“DB access goes through Prisma except src/legacy/ and src/db/raw/.” Specific exception that prevents the model from rewriting raw queries it sees.
“TOKEN_CACHE in src/middleware/auth.ts is deprecated; do not extend it.” Stops the model from “helpfully” adding to deprecated code.
“No --no-verify on commits. Pre-commit hooks are load-bearing.” Specific anti-pattern; model will use –no-verify to “fix” hook failures otherwise.
“No mocking the DB in integration tests (see /docs/testing.md).” Specific anti-pattern with reason via link.
“Don’t leave -- TODO in migration Down blocks; it breaks rollback tests.” Specific, with the specific test that fails.
“Architecture: /docs/architecture.md (read on demand).” Pointer, not content.
“What I am NOT going to remind you about: style, test running, folder layout.” Sets boundaries — saves the model time looking for guidance that isn’t there.

The lesson

A great CLAUDE.md is the smallest possible set of load-bearing invariants the model cannot derive from the code, plus the anti-patterns it would otherwise reach for, plus explicit non-goals so it stops looking.

If you can’t answer the question “what would the model otherwise get wrong?” for a line — cut it.

The math: a 500-byte section, in a CLAUDE.md, at 20 turns a session, 5 sessions a week, 50 weeks a year, across 50 engineers — that’s 125 megabytes of model input per year per section. Bloat isn’t aesthetic; it’s measurable.
